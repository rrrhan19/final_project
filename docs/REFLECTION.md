# 專題心得（作業 #5 心得 / #6 AI 是能力放大器）

## 動機
查資料時看到「2024 年詐騙財損 502 億、12 萬件」，數字背後是一個個受傷的人。我們想做一個**任何人都用得起**的反詐工具，把「看穿詐騙」這件需要經驗的事，變成貼上訊息就能做到。

## 我們做了什麼
一個完整的反詐平台：Vue 前端 + Node（模擬遊戲）+ Python FastAPI（自訓詐騙模型 + Google Gemini + 統計）+ PostgreSQL，資料用爬蟲從**警政署 165 官方開放資料**收集，可部署於 render / AWS，並備有 Jetson 邊緣端程式。

## AI 是能力放大器（而非取代）
這是我最大的體會。一個學生團隊要獨力完成「全棧 + 機器學習 + 邊緣裝置」幾乎不可能，但：
- **自訓模型負責判斷、Gemini 負責解釋**——AI 補足我們不擅長的部分，讓產品「看得懂」。
- **AI 協作開發**——從架構設計、踩坑除錯到測試，AI 像一個隨時在線的資深隊友，把我的學習速度與產出放大數倍。
- 重點是：**方向、取捨、誠信仍由人決定**。例如顧問式檢查抓出「統計用了示意假數據卻標成官方」，是我們親手修正的——AI 放大產出，但品質與誠實要自己守住。

## 最難的關卡
1. **資料不平衡**：官方資料幾乎全是詐騙樣態，模型一度把正常訊息誤判詐騙率高達 69%。補正常樣本 + 模型主導的 ensemble 後降到 7.7%。這讓我真正理解「資料品質決定模型」。
2. **誠信**：把示意數字當官方資料是信用地雷。我們改用可引用的真實官方統計並標明口徑限制——**做產品，誠實比好看更重要**。
3. **過擬合的假象**：訓練集準確率近 100% 毫無意義，改用 held-out 測試集才看到真實表現。

## 學到的事
- 端到端把一個想法做成可跑、可部署、可被評估的產品。
- 機器學習的指標素養（Recall/誤報率/校準），而非只看 accuracy。
- 守法與倫理（robots.txt、去個資、資料出境揭露）也是工程的一部分。
- 用 AI 放大自己，但對結果負責的是人。

## 如果重來
會更早設計評估與資料來源治理，而不是先把功能做完才回頭補誠信與量測。
