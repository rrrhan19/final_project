## 研究動機與背景
近年詐騙犯罪在台灣急速惡化：依內政部刑事警察局統計，2024 年詐騙案件達 122,805 件、財損超過 502 億元，平均每天逾 336 件，五年成長逾四倍（註：2024 起採 165 打詐儀錶板新口徑）。詐騙手法不斷翻新（假投資、假冒公務機關、解除分期、釣魚簡訊等），一般民眾難以即時辨識。我們希望運用 AI 與公開資料，打造一個人人可用的反詐輔助平台，降低受害風險。

## 研究目的
1. 建立可即時判斷訊息／網址是否為詐騙、並**解釋理由**的偵測服務。
2. 以模擬遊戲提升民眾辨識詐騙的能力（教育）。
3. 將官方詐騙資料視覺化，呈現趨勢與類型分布（資料分析）。
4. 驗證「AI 作為能力放大器」：以自訓模型＋大型語言模型協作，讓小團隊產出完整產品。

## 系統功能設計
- **詐騙偵測**：輸入訊息/網址 → 自訓分類器（TF-IDF＋LogisticRegression）給詐騙機率、Google Gemini 給理由解釋、RAG 找相似歷史案例，三者 ensemble 並逐層降級；回傳 verdict／信心／理由／相似案例。
- **模擬遊戲**：情境問答，後端批改並給解說（伺服端判分，不洩答案）。
- **詐騙統計**：官方年度趨勢 + 收集樣本之類型分布，標明資料來源與口徑。
- **資料收集**：爬蟲自警政署 165 開放資料收集詐騙案例，去個資後入庫供模型與 RAG 使用。

## 開發環境與技術
- 前端：Vue 3 + Vite；後端：Node.js(Express) + Python FastAPI；資料庫：PostgreSQL。
- ML：scikit-learn；LLM：Google Gemini；爬蟲：Selenium/Playwright + 政府開放資料。
- 部署：render（主）/ AWS（備）；加分：NVIDIA Jetson Orin Nano 邊緣端。
